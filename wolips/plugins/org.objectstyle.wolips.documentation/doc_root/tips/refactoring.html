Refactoring with WOLips

Eclipse has great built-in support for refactoring. For WO these are not enough...

Move and/or Rename Enterprise Objects

  1. Move/Rename the java file with the Eclipse built-in Java refactoring functionality.
  2. (You have to know the name of the entity linked to this class.) Open EOModeler and set the entity's class to be the new fully qualified class name (or edit the entity's *.plist to do the same thing).

Move Components

  1. Move the java file with the Eclipse built-in Java refactoring functionality.

If the component used to be in the default (no) package and its accessor mothods are protected, you have a problem. This problem has a solution.

WOBuilder likes to add (sub) components with their fully qualified names. This is not necessary. Using just the class name works fine.

Rename Components

  1. Rename the java file with the Eclipse built-in Java refactoring functionality.
  2. Rename the components other files: *.api, *.html, *.wod & *.woo
  3. Find all instances of the old name in *.wod files and replace it with the new name.
