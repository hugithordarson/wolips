Profiling WebObjects Applications

WOLips integrates with jMechanic ( http://jmechanic.sourceforge.net/ ).

jMechanic has nothing to do with WebObjects or WOLips. It is an Eclipse Java IDE plug-in providing profiling capabilities. As such, you as a WOLips user, can use it.

WOLips includes a profiling plug-in that extends jMechanic. Unless the actual jMechanic plug-in is also installed this profiling plug-in will do nothing. When both the jMechanic and the WOLips profiling plug-ins are present there will be an additional launch configuration type - Profiled WOApplication. (As well as the standard jMechanic features.) You can use this the same way you use plain WOApplication configurations. Running applications based on Profiled WOApplication templates is the same as running applications based on WOApplication templates. Debugging is different. Only when you debug do you get the jMechanic features (or use the extra memory and CPU resources). It should be feasible to base all WOLips launch configurations on the Profiled WOApplication template.
