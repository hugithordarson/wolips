Installing WOLips

Requirements

  • Eclipse 2.1.
  • A complete and working WebObjects 5.1/5.2 Development installation. (Setup WebObjects on Windows)
  • JDK 1.3.1 or above - 1.4.1 is ok.
  • If you're on Macintosh you must use at least OS X 10.2.

Installing

If you're installing WOLips for the first time or if you are upgrading from a release prior to the final release of WOLips 1.0; follow these instructions.

  • Quit Eclipse if you have it running.
  • If you have any previous version of WOLips installed then uninstall that first.
  • Download the latest WOLips release from SourceForge. **
  • Unpack the downloaded file. You should end up with a folder named something like WOLips_x.y.z. This folder should in turn contain three folders: features, plugins and optional.
  • Copy the contents (not the *.jar files) of the features and plugins folders to C:\eclipse\features and C:\eclipse\plugins respectively. (The paths may be different on your system.)
  • Read the README.html file in the folder named optional. There may be additional plug-ins or featues to install.
  • Restart Eclipse. (On Macintosh it's 'normal' that you have to launch Eclipse twice after changes to the plug-ins and features.) When you start Eclipse with a new or updated feature you have to activate it with the Update Manager. It should appear automatically when Eclipse is started. If not; you’ll find it under Help > Software Updates > Update Manager.
  • Read the What's New section of the WOLips on-line documentation. It may contain information regarding necessary actions with this particular version.
  • The WOLips perspective should be reset after updates.

The WOLips_x.y.z folder mentioned above also contains a site.xml file. This is in effect an update site. The installation could have been performed using the Update Manager. You may read about "Updating features with the update manager" in the Workbench User Guide.

Both the Spindle (Tapestry) and VeloEdit (Velocity) plug-ins have good instructions for using the Eclipse update manager. The official WOLips update site URL is: http://woproject.sourceforge.net/update/

Updating

Once you have the final release of WOLips 1.0 installed you can perform future updates using the built-in Update Manager. Eclipse is now configured to check for updates at the WOLips update site.

  • Select the workbench menu option: Help > Software Updates > New Updates
  • ...

It is of course also possible to update WOLips using brute force - simply delete and replace the installed features and plug-ins.

 

** It is also possible, and not difficult, to build from source.
